Product architecture

Local-first football rooms with a self-custodial wallet layer.

MatchMesh is built for fans watching football together in places where connectivity can be uneven. The app keeps room entry, chat, points, live match context, creator tips, and wallet actions in one loop.

Wallet behavior

  • The wallet belongs to the current user profile and exposes a receive QR for the active account.
  • Users start with one Spending pocket and can add more pockets for pools, creator payouts, or matchday costs.
  • Transfers create a wallet receipt immediately and attempt WDK execution when a native wallet, RPC, and valid recipient are available.
  • The devnet mock USDT mint is configured by default, so wallet transfers use SPL tokens. If the mint is removed, devnet SOL is used as a fallback proof path.
  • Recovery, import, export, rotation, and fund removal are hidden behind a Recovery & keys drawer so sensitive actions stay deliberate.

Low connectivity path

MatchMesh keeps the critical experience small: room code, local room log, cached fixture state, local assistant responses, and queued wallet receipts. Full P2P discovery is enabled when the native runtime is started with Hyperswarm support.

App flow iteration

  • Create or rename a room.
  • Copy the invite link or MESH code.
  • Join from another window, phone, or browser profile.
  • Send messages and watch points update.
  • Use the live pitch view for score, players, and match events.
  • Send a payment, tip the top commentator, or receive USDT from a QR code.
